Отличии git rebase от git merge?

Git rebase сжимает все изменения в один пакет и этот новый пакет объединяет целевую ветку.

Затем он перемещает выполненную работу из одной ветки в другую, обычно в главную ветку.

В процессе rebase выравнивает историю, удаляя ненужные записи.

С другой стороны, Git merge изменяет только целевую ветку и создает коммит, сохраняя историю исходной ветки.

Что такое Git fetch?

Оно скачивает всееее файлы, всююююю историю и встраивает в вашу локальную папку

Отличии git fetch от git pull?

git fetch скачивает из удаленного все содержимое, но не изменяет состояние локальной папки.

a git pull скачивает из удаленного все содержимое и

сразу пытается изменить состояние локального файла,

чтобы оно соответствовало этому содержимому.
